# GSRS — Project Bootstrap

STATUS: ACTIVE

PURPOSE:
A calm, elegant, continuity-first gemstone catalog and presentation system designed for long-term survivability, graceful evolution, and low operational friction.

CORE PRINCIPLES:
• Stable shell philosophy
• Evolution without user shock
• Grace survivability rule
• Anti-scope-creep enforcement
• Photography-first presentation
• Collector/gallery aesthetic over ecommerce aesthetic
• Simple systems survive

STACK:
• Frontend: Next.js + Tailwind
• Database: PostgreSQL
• Hosting: Render
• Source Control: GitHub
• Dev Environment: VS Code
• OS Direction: Ubuntu Linux

VISUAL LANGUAGE:
• Dark luxury aesthetic
• Museum/archive influence
• Cinematic gemstone presentation
• Elegant typography
• Calm transitions
• High negative space
• Gemstone as protagonist

CURRENT STATUS:
• UI foundation started
• Brand direction aligned with Gemini mp4 aesthetic
• Access system considered legacy/non-authoritative
• PostgreSQL available locally
• Local-first deployment strategy active

ARCHITECTURAL RULES:
• Do not overengineer phase 1
• No premature ecommerce
• No microservices
• No distributed systems
• No enterprise abstractions
• Inventory workflow simplicity is priority

THREAD POLICY:
Threads are temporary execution surfaces.
Persistent markdown files are authoritative memory.
After saturation:
thread → extraction → persistence → delete thread
